# メール送信前チェック（email-check）

メール送信エージェント（Claude Code + Gmail MCP、ブラウザエージェント、CRMのシーケンス機能）が作った文面を、送信前に独立した目で検査する。

## 原則

- **書いた本人に採点させない。** 文面をこのメインコンテキストで生成した場合は、必ず `email-reviewer` サブエージェント（Agentツール、subagent_type: email-reviewer）に検査を委譲する
- **C（Critical）が1件でもあれば「送信不可」と明言する。** 「ご確認ください」で曖昧に流さない
- **送信操作は人間の承認後のみ。** チェック合格≠自動送信
- チェックリストの全項目（C/W/S重大度別）は `references/checklist.md` を読む

## 手順

1. 入力を集める:
   - 文面: 会話に貼られたもの / Gmail下書き（list_drafts→該当draft）/ ブラウザ画面のテキスト
   - 文脈: 宛先アドレス・相手の会社名/氏名/役職・関係性（初回営業/既存/紹介経由/フォロー）・差し込み元データ（CRM情報）
   - 不明な文脈は推測せず「未確認」として扱い、結果に明記する
2. `email-reviewer` サブエージェントに「文面＋宛先＋文脈」を渡して検査させる
3. 結果を報告する: 判定（送信可 / 送信不可 / 条件付き可）→ 指摘（[C/W/S] 箇所→問題→直し方）→ 修正版全文
4. A/Bテスト運用がある場合は件名の代替案を2本付け、記録先（スプレッドシート等）に宛先・件名バージョン・送信日時・反応を追記する
5. ユーザー承認後にのみ送信系の操作に進む

## 外部エージェントへの転用

ブラウザエージェント等にこの検査を組み込む場合は、`references/checklist.md` のC/W/S表をそのまま指示文に貼り、末尾に「送信前に必ずこの検査を行い、Cが1件でもあれば送信せず報告すること」と付ける。
