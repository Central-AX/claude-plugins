# メール送信前チェックリスト（C/W/S重大度制）

外部エージェントの指示文にもそのまま転用できる完全版。

## C: 事故系（1件で送信不可）

| # | 項目 | 見るポイント |
|---|------|------------|
| C1 | 宛先と文面の不一致 | toのドメイン ↔ 本文の会社名・人名。過去スレッド・署名との整合 |
| C2 | 差し込み残骸 | {{company}}、[会社名]、●●、〇〇様、(株)未展開、テンプレ文のまま |
| C3 | 他社情報の混入 | 別会社向け文面の固有名詞・数値・案件名・添付の残留 |
| C4 | リンク/添付の取り違え | リンク先が宛先会社向けの資料か。リンク切れ（WebFetchで生存確認） |
| C5 | To/Cc/Bcc事故 | 一斉送信の宛先がToに並んでいないか |

## W: 信頼毀損系

| # | 項目 | 見るポイント |
|---|------|------------|
| W1 | 敬称 | 会社名には「様」（添付ファイル名も含む）。「様様」「社長様」等の二重敬称。役職と氏名の順序 |
| W2 | 事実主張の根拠 | 相手企業に関する記述（業種・ニュース・導入実績）の裏取り。数値の出典。不確かなら断定を弱める |
| W3 | 特定電子メール法 | 営業メールなら: 送信者の氏名/名称・住所等の表示、オプトアウト（配信停止）導線、同意・取引関係の有無 |
| W4 | スパムシグナル | 件名の【】!?乱用、煽り語、本文のURL過多 |

## S: 品質系

| # | 項目 | 見るポイント |
|---|------|------------|
| S1 | 件名 | 20字前後で用件が伝わるか。開封したくなるか |
| S2 | 構成 | 第一段落で「誰が・なぜ・何を」。CTAは1つ。日程提示は具体案で |
| S3 | トーン | 関係性に合うか。AI定型文臭（「ぜひ」「いかがでしょうか」連発、過剰な箇条書き）の除去 |
| S4 | 長さ | スマホ1.5スクロール以内が目安 |
